# Claude Code Team Workflow

## Workflows

### Feature Development

```
/plan ‚Üí /implement ‚Üí /review ‚Üí /commit
```

| Step | Skill        | What it does                                        | When to use         |
| ---- | ------------ | --------------------------------------------------- | ------------------- |
| 1    | `/plan`      | Read Notion ticket ‚Üí analyze codebase ‚Üí create plan | Medium/large tasks  |
| 2    | `/implement` | Code based on plan, follow standards, run tests     | All features        |
| 3    | `/review`    | Review code quality ‚Üí fix issues ‚Üí verify           | Before commit       |
| 4    | `/commit`    | Lint ‚Üí test ‚Üí commit ‚Üí update Notion                | After review passes |

### Bug Fixes

```
/fix ‚Üí /review ‚Üí /commit
```

| Step | Skill     | What it does                                               | When to use         |
| ---- | --------- | ---------------------------------------------------------- | ------------------- |
| 1    | `/fix`    | Reproduce ‚Üí debug ‚Üí fix ‚Üí regression test                  | All bug fixes       |
| 2    | `/review` | Review code quality ‚Üí fix issues ‚Üí verify                  | Before commit       |
| 3    | `/commit` | Lint ‚Üí test ‚Üí commit ‚Üí update Notion                       | After review passes |

## How to Use

### Bug fix:

```bash
/fix [Notion ticket link or bug description]
/review
/commit
```

### Simple feature (single file):

```bash
/implement [task description or Notion link]
/review
/commit
```

### Complex feature (multiple files):

```bash
/plan [Notion ticket link]
/implement plans/task-name.md
/review
/commit
```

### Full workflow examples:

**Feature workflow:**
```
/plan ‚Üí creates plans/task.md + updates Notion ‚Üí prints "Next: /implement plans/task.md"
  ‚Üì
/implement plans/task.md ‚Üí follows plan checklist ‚Üí marks [x] ‚Üí prints "Next: /review"
  ‚Üì
/review ‚Üí project checks + code-reviewer agent ‚Üí fixes issues ‚Üí prints "Next: /commit"
  ‚Üì
/commit ‚Üí verify review ‚Üí branch check ‚Üí sync plan ‚Üí stage ‚Üí commit ‚Üí Notion ‚Üí push ‚Üí done
```

**Bug fix workflow:**
```
/fix [ticket] ‚Üí reproduce ‚Üí debug ‚Üí fix ‚Üí regression test ‚Üí prints "Next: /review"
  ‚Üì
/review ‚Üí project checks + code-reviewer agent ‚Üí fixes issues ‚Üí prints "Next: /commit"
  ‚Üì
/commit ‚Üí verify review ‚Üí branch check ‚Üí stage ‚Üí commit ‚Üí Notion ‚Üí push ‚Üí done
```
