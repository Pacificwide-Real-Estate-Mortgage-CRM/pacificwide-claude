# /sequential-thinking — Step-by-Step Analysis

Structured problem-solving via reflective thought sequences with dynamic adjustment and revision.

## Usage

When to use: complex decomposition, adaptive planning, unclear scope, hypothesis-driven debugging, multi-step solutions.

## Workflow

1. **Start** with loose estimate: `Thought 1/5: [Initial analysis]` — adjust total dynamically
2. **Each thought**: build on previous, one aspect per thought, state assumptions and uncertainties
3. **Adjust dynamically**: expand/contract total, revise invalidated thoughts, branch for alternatives
4. **Revise**: `Thought 5/8 [REVISION of Thought 2]: [Corrected understanding]`
5. **Branch**: `Thought 4/7 [BRANCH A]: ...` vs `[BRANCH B]: ...` — compare, converge with rationale
6. **Hypothesize**: `[HYPOTHESIS]` → `[VERIFICATION]` — iterate until verified
7. **Complete**: `Thought N/N [FINAL]` — only when solution verified and confidence achieved

## Rules

- Explicit mode: show thought markers when complexity warrants visible reasoning
- Implicit mode: apply methodology internally for routine problems
- Never claim final without addressing all critical aspects
